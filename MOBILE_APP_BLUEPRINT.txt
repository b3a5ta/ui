================================================================================
  1. EXECUTIVE SUMMARY
================================================================================

Application: NuViO369 CRM (Interview CRM)
Developer: Futuremug
Purpose: Recruitment/Interview pipeline management system with AI-powered
         resume matching and automated calling capabilities.

The web frontend is a server-side rendered (SSR) application built with:
  - Rust (Spin SDK 3.0 / Wasm) as the HTTP handler
  - Askama templates for HTML rendering
  - Vanilla JavaScript (ES6+) for client-side interactivity
  - Tailwind CSS 3.4.1 for styling
  - No frontend framework (React, Vue, etc.) - pure JS

The application has 6 core modules:
  1. Dashboard - Overview statistics
  2. Candidates (Leads) - Full CRUD + AI calling + bulk import
  3. Contacts - Full CRUD
  4. Interactions - Full CRUD with entity linking
  5. Opportunities - Full CRUD + AI calling + records
  6. Resume Matcher - AI-powered resume-to-JD matching with bulk calls

API BASE URL: Configured dynamically

AUTHENTICATION MECHANISM:
â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€â”€
  All authenticated endpoints require:
    Authorization: Bearer {JWT_TOKEN}
    Content-Type: application/json

  Token storage (web): localStorage keys 'token', 'refresh_token', 'user'
  Token storage (mobile): Tauri secure store / platform keychain

  JWT Claims structure:
    {
      "role": "Super Admin" | "Staff 1" | "Staff 2" | "Staff 3",
      "allowed_services": ["candidates", "contacts", "interactions", "opportunities"],
      "exp": <timestamp>,
      "sub": "<user_id>"
    }

  Global 401 interceptor: On any 401 response, attempt token refresh.
  If refresh fails, force logout and redirect to login.

  Session timeout: 15-minute inactivity auto-logout (web only - mobile should
  use background app state detection instead).

POST /api/auth/login
  Purpose:     User authentication
  Auth:        None (public)
  Request:     { "email": string, "password": string }
  Response:    {
                 "token": "JWT string",
                 "refresh_token": "string",
                 "user": { "id": "UUID", "name": string, "email": string, "role": string }
               }
  Errors:      401 Invalid credentials

POST /api/auth/register
  Purpose:     New user registration (admin creates users)
  Auth:        None (public)
  Request:     {
                 "name": string (required),
                 "email": string (required, email format),
                 "password": string (required, min 6 chars),
                 "confirm_password": string (must match password),
                 "role": string
               }
  Response:    Success message or error
  Errors:      409 Email already exists, 400 validation errors

POST /api/auth/refresh
  Purpose:     Refresh expired JWT access token
  Auth:        None (but requires valid refresh_token in body)
  Request:     { "refresh_token": string }
  Response:    {
                 "token": "new JWT string",
                 "refresh_token": "optional new refresh token"
               }
  Errors:      401 Invalid/expired refresh token

  Layout: Sidebar + Header + Controls bar + Data table + Pagination

  CONTROLS BAR:
    - Search input (searches name, email, phone)
    - "Add Candidate" button (primary, links to /candidates/add)
    - Status filter dropdown:
      All Statuses, New, Contacted, Call Back Requested,
      Screening Completed, Not Interested, Busy, Not Answered
    - Year filter dropdown (current year - 15 years back)
    - Month filter dropdown (enabled only when year is selected)
    - From Date picker (with validation)
    - To Date picker (with validation)
    - Divider
    - Export to Excel button
    - Bulk actions panel (appears when items selected):
      "{N} selected" + Delete button

  TABLE COLUMNS:
    1. Checkbox (bulk select)
    2. Name + Email (name is clickable link to view page)
    3. Phone
    4. Skills (truncated to 3 with "+N" indicator)
    5. Resume (PDF icon if resume_url exists, downloadable)
    6. Status (inline dropdown - change triggers confirmation modal + API PUT)
    7. Actions:
       - Call (green phone icon, disabled if no valid +91 phone)
       - View (blue eye icon -> /candidates/view?id=)
       - Edit (purple pen icon -> /candidates/edit?id=)
       - Records (purple clipboard -> /candidates/record?id=)
       - Delete (red trash icon -> confirmation modal)

  PAGINATION:
    - "Showing X-Y of Z records"
    - Page size selector: 10, 25, 50, 100
    - Page number buttons with ellipsis
    - Prev/Next chevron buttons

  SORTING:
    - Clickable column headers: Name, Phone, Status
    - Toggle asc/desc with sort icon indicator

  FILTERING LOGIC (client-side):
    - All filters combine with AND logic
    - Search: partial match on name, email, phone (case-insensitive)
    - Status: exact match
    - Date range: compares created_at field (substring YYYY-MM-DD)
    - Year/Month cascading: sets from/to date automatically

  DATE VALIDATION:
    - No future dates allowed
    - No dates older than 15 years
    - From date cannot be after To date
    - Visual error state (red border) on invalid dates
    - Validation toast notification

  INLINE STATUS CHANGE:
    - Dropdown in table cell changes status
    - Triggers confirmation modal showing current -> new status
    - On confirm: PUT /api/candidates/{id} with full candidate object
    - On cancel: reverts dropdown to original value

  CALL INITIATION:
    - Phone validation: must match /^(\+91|91)\d{10}$/
    - Shows call confirmation modal with candidate details
    - POST /api/candidates/{id}/call
    - On success: updates status to "Contacted" locally
    - Button shows calling animation while in progress

  EXPORT:
    - Sends current filter state as query params
    - Downloads .xlsx blob file
    - Filename includes date range if filtered

  BULK DELETE:
    - Iterates over selected IDs, sends DELETE for each
    - Refreshes data after completion

    - Position Applied For (text)
    - Notes/Comments (textarea, full-width)
    - Hidden: import_source = "manual"

  ACTIONS:
    - Cancel button -> navigate to /candidates
    - Create Candidate button -> createLead() function

  SUBMISSION LOGIC:
    1. HTML5 form validation (reportValidity)
    2. Extract resume file separately
    3. Convert numeric fields to proper types
    4. Split skills string into array
    5. Set empty strings to null
    6. POST /api/candidates (create candidate)
    7. If resume provided: POST /api/attachments (upload as base64)
    8. Success toast -> redirect to /candidates after 1.5s

  ERROR HANDLING:
    - 409: "Phone number already exists" - highlights phone field
    - LinkedIn URL error: highlights linkedin field
    - Resume upload failure: shows warning but doesn't fail creation
